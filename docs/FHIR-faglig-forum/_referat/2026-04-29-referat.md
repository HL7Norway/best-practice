### Fordeler med FHIR server

#### Mønster for FHIR-server

* klinisk fagsystem deleer via FHIRserver
* FHIR server som primærlager
* etc

#### Beskrivelse av mønstrene

* Fagsystem deler via FHIRserver
* Enklere å legge data inn i FHIR server enn å bruke applikasjonen direkte for legacy systemer
* Forutsetter at fagsystemet støtter push av data, transformering av data til FHIR

#### Flere applikasjoner deler inn i FHIR server

* Kontroll på tilgang og governance

#### Konsolidering av data i FHIR server

* Les og skriv til FHIR server fra alle applikasjonene

#### FHIR server som primærlager

* Egenbehandlingsplaner som mange applikasjoner skal oppdatere og lese
* Grunndataregister

#### FHIR server for les og søk

### FHIR fasade mønster

* Virkemåte så er visjonen smartapper kan integreres i vertssystemer

  * Idag er det mye skreddersøm som man ønsker å kutte
* Tilgangskontroll
